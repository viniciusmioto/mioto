# Publication Structure and Generation Flow

This repository uses Hugo with the Hugo Blox Builder/Academic CV theme. Publications are created as a Hugo section under `content/publication/`, and they are rendered into static HTML pages under `public/publication/`.

## Content structure

- `content/publication/` is the main publications section.
- Each publication is stored in its own folder under `content/publication/`, for example:
  - `content/publication/gas2025/`
  - `content/publication/brasnam2025/`
  - `content/publication/conpet2021/`
- Each publication folder contains an `index.md` file with page front matter and optional content.
- A publication page can also include local assets inside the same folder, such as `featured.jpg`, PDFs, or other attachments.
- `content/publication/_index.md` defines the publication section landing page and controls how the publication list is rendered.

## Typical publication markdown file

The publication markdown file uses Hugo front matter in YAML format.
Example: `content/publication/gas2025/index.md`

Important front matter fields include:

- `title`: page title shown in the generated HTML and metadata.
- `authors`: a list of author names, usually matching profiles in `content/authors/`.
- `date`: the publication or paper date.
- `hugoblox.ids.doi`: DOI identifier for the publication.
- `publishDate`: page publish date used by Hugo/Blox.
- `publication_types`: taxonomy list for publication type, e.g. `['paper-conference']`.
- `publication`: full publication venue name.
- `publication_short`: shorter venue text or abbreviation.
- `abstract`: the publication abstract.
- `summary`: optional short summary used by list cards.
- `tags`: topic tags such as `Software Engineering`.
- `featured`: whether the publication appears in featured widgets.
- `image`: featured image settings.
- `projects`: associated project page references.
- `slides`: optional slides association.

The body of the markdown file is rendered as page content below the front matter. In `gas2025` it is:

`9th International Workshop on Games and Software Engineering - Co-located with ICSE 2025, Ottawa, Canada, 2025.`

## Hugo configuration for publications

Relevant configuration is in `config/_default/hugo.yaml`.

Key settings:

- `taxonomies:`
  - `author: authors`
  - `tag: tags`
  - `publication_type: publication_types`

This means Hugo treats `publication_types` as a taxonomy, generating pages like:
- `public/publication_types/paper-conference/`

Other important Hugo settings:

- `outputs.section: [HTML, RSS]` enables section output pages.
- `permalinks` and other website-level rules are defined in this config.

## Generated HTML output

Hugo builds each publication page into `public/publication/<slug>/index.html`.
For example:
- `public/publication/gas2025/index.html`

The stack for generated pages includes:

- HTML page shell generated by Hugo and the active theme/module templates.
- CSS loaded from `public/css/themes/sky.min.css`.
- JS loaded from `public/js/hugo-blox-en.min.js` and `public/js/hb-head.min.*.js`.
- Optional Alpine.js library loaded from `public/dist/lib/alpinejs/cdn.min.*.js`.

The output HTML page contains:

- metadata tags (`title`, `description`, `og:*`, `twitter:*`, canonical URL).
- a site sidebar and navigation links generated from the publication section.
- rendered publication front matter fields like abstract, publication venue, type, tags, and authors.
- optional related links such as author profiles, tag pages, and publication type pages.
- the page body rendered from the Markdown content.

## Section and taxonomy pages

Generated pages include:

- `public/publication/index.html` — the section listing all publications.
- `public/publication/<slug>/index.html` — each publication detail page.
- `public/publication_types/<type>/index.html` — taxonomy pages for publication types.
- `public/tags/<tag>/index.html` — tag listing pages.
- `public/author/<author>/index.html` — author profile pages with publication references.

## How the repository supports imports

The repo includes a workflow in `.github/workflows/import-publications.yml`.

That workflow can import publications from a BibTeX file into `content/publication/` using the Hugo Academic CLI:
- `academic import publications.bib content/publication/ --compact`

This means publication folders and `index.md` files may be generated from BibTeX, then Hugo renders them as pages.

## Summary

1. Publication source files are under `content/publication/<slug>/index.md`.
2. Each file is a Hugo page with YAML front matter and optional body content.
3. Hugo config maps `publication_types` to a taxonomy and enables section outputs.
4. Hugo builds static HTML under `public/publication/` and related taxonomy/tag pages.
5. JavaScript assets such as `hugo-blox-en.min.js` and `hb-head.min.*.js` are included in each generated page.

This flow connects the Markdown content, Hugo section/taxonomy configuration, and the final generated HTML + JS output.
